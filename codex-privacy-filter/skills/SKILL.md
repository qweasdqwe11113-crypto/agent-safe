# 隐私过滤

当文本中可能包含 API Key、Bearer Token、密码、邮箱、手机号或内部敏感标识符时，可以使用这个技能进行检查和脱敏。

## 这个插件是做什么的

这个插件为 Codex 项目提供一个本地的隐私过滤工作流，用于在文本被复制到日志、报告、提示词或共享输出之前，先进行敏感信息清洗。

## 什么时候使用

当你需要进行以下操作时，可以使用这个技能：

- 对日志或终端输出中的敏感信息进行脱敏
- 在分享命令内容之前先清洗其中的敏感字段
- 处理复制出来的配置文件内容
- 在 Bug 报告或截图转文字内容中隐藏凭证信息
- 在继续传递文本之前降低隐私泄露风险

## 如何使用

1. 收集可能包含敏感信息的文本。
2. 对文本内容或文件内容运行脱敏脚本。
3. 检查脱敏结果，确认是否存在误报或漏报。
4. 将清洗后的结果用于后续的 Codex 步骤。

## 脚本位置

当前实现位于 `./scripts/redact.py`。

## 说明

这是一个初始原型版本。当前实现主要基于模式匹配和正则规则，因此既可能误伤普通文本，也可能漏掉少见的密钥格式。在把结果当成最终安全输出之前，仍然需要人工复查。
